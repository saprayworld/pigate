🔌 PiGate Backend API Endpoints

แผนผัง API Spec สำหรับการเชื่อมโยงระบบ Dashboard & Firewall (ออกแบบสำหรับ Frontend)
📊 1. System & Dashboard API (การดึงสถานะ Real-time)
GET /api/system/status ดึงสถิติ CPU, Memory, อุณหภูมิ และปริมาณการรับส่งข้อมูล (WAN Traffic) สำหรับ Dashboard
[ คลิกดูรายละเอียด ]
Response (200 OK)
{ "hostname": "PiGate-RPI5", "uptime": "2d 14h 32m", "firmware": "v1.0.0", "cpu": { "usage_percent": 14.5, "cores": 4, "temperature_celsius": 48.5 }, "memory": { "total_bytes": 8160436224, "used_bytes": 1224065433, "percent_used": 15.0 }, "storage": { "total_bytes": 62283022336, "used_bytes": 25732104192, "percent_used": 41.3 }, "traffic": { "interface": "wlan0", "in_bps": 4200000, "out_bps": 1100000 } }
GET /api/system/logs ดึงประวัติการทำงานและ log การบล็อกข้อมูลของ nftables (รองรับการกรองคำค้นหา)
[ คลิกดูรายละเอียด ]
Query Parameters
Parameter Type Required Description
limit integer Optional จำนวนรายการที่ต้องการดึง (ค่าเริ่มต้น 50)
search string Optional คำค้นหา Source IP หรือพอร์ต
Response (200 OK)
{ "logs": [ { "time": "2026-06-05T16:20:11Z", "src_ip": "192.168.1.105", "dst_ip": "8.8.8.8", "proto": "UDP", "port": 53, "action": "PASS" }, { "time": "2026-06-05T16:19:45Z", "src_ip": "192.168.1.132", "dst_ip": "203.0.113.5", "proto": "TCP", "port": 23, "action": "DROP" } ] }
POST /api/system/action สั่งงานระบบ เช่น reboot หรือ shutdown บอร์ด Raspberry Pi
[ คลิกดูรายละเอียด ]
Request Body (JSON)
{ "action": "reboot" // หรือ "shutdown" }
Response (200 OK)
{ "success": true, "message": "System is rebooting now..." }
🌐 2. Interfaces API (การจัดการพอร์ต LAN/WiFi)
GET /api/interfaces ดึงรายการข้อมูลอินเตอร์เฟสทั้งหมดในระบบ (Physical & Virtual)
[ คลิกดูรายละเอียด ]
Response (200 OK)
[ { "name": "eth0", "alias": "LAN_Internal", "type": "ethernet", "ip_address": "192.168.1.1", "netmask": "255.255.255.0", "mode": "static", "admin_access": ["PING", "HTTP", "SSH"], "status": "UP" }, { "name": "wlan0", "alias": "WAN_WiFi", "type": "wifi", "ip_address": "10.0.0.45", "netmask": "255.255.255.0", "mode": "dhcp", "admin_access": ["PING"], "status": "UP", "wifi_settings": { "ssid": "MyHome_5G", "signal_percent": 85 } } ]
PUT /api/interfaces/{name} บันทึกค่าการแก้ไขอินเตอร์เฟส (เช่น แก้ alias, โหมด IP, Wi-Fi หรือ สิทธิ์ Admin Access)
[ คลิกดูรายละเอียด ]
Request Body (JSON)
{ "alias": "WAN_WiFi", "mode": "dhcp", // "dhcp" หรือ "static" "ip_address": "", // หากเป็น static ต้องส่งค่านี้มาด้วย "netmask": "", "admin_access": ["PING"], // รายการสิทธิ์ เช่น "PING", "HTTP", "HTTPS", "SSH" "wifi_settings": { // ส่งเฉพาะอินเตอร์เฟสที่เป็น wifi "ssid": "MyHome_5G", "psk": "mypassword123" } }
Response (200 OK)
{ "success": true, "message": "Interface configuration updated successfully" }
POST /api/interfaces/wlan0/scan สั่งสแกนสัญญาณ Wi-Fi บริเวณรอบข้างเพื่อนำมาแสดงรายการ SSID ให้ผู้ใช้เลือก
[ คลิกดูรายละเอียด ]
Response (200 OK)
[ { "ssid": "MyHome_5G", "signal": 85, "security": "WPA2-PSK" }, { "ssid": "Cafe_Free_WiFi", "signal": 40, "security": "OPEN" } ]
🔥 3. Firewall Policy API (การจัดการกฎการกรองข้อมูล)
GET /api/policies ดึงรายการกฎนโยบายไฟร์วอลล์ทั้งหมด จัดเรียงตามลำดับความสำคัญ
[ คลิกดูรายละเอียด ]
Response (200 OK)
[ { "id": 1, "seq": 1, "name": "Allow-Web-Out", "src": "LAN_Network", "dst": "ALL (Internet)", "service": ["HTTP (80)", "HTTPS (443)"], "action": "ACCEPT", "log": false, "status": true }, { "id": 2, "seq": 2, "name": "Block-BitTorrent", "src": "LAN_Network", "dst": "ALL", "service": ["BitTorrent_Ports"], "action": "DROP", "log": true, "status": true } ]
POST /api/policies สร้างกฎไฟร์วอลล์ใหม่
[ คลิกดูรายละเอียด ]
Request Body (JSON)
{ "name": "Allow-SSH-In", "src": "192.168.1.100", "dst": "LAN_Internal", "service": ["SSH (22)"], "action": "ACCEPT", "log": true, "status": true }
Response (201 Created)
{ "success": true, "id": 3, "message": "Policy created successfully" }
PUT /api/policies/{id} แก้ไขกฎการทำงานเดิม
[ คลิกดูรายละเอียด ]
Request Body (JSON)
{ "name": "Allow-Web-Out-Updated", "src": "LAN_Network", "dst": "ALL (Internet)", "service": ["HTTP (80)", "HTTPS (443)"], "action": "ACCEPT", "log": true, "status": true }
Response (200 OK)
{ "success": true, "message": "Policy updated successfully" }
DELETE /api/policies/{id} ลบกฎไฟร์วอลล์
[ คลิกดูรายละเอียด ]
Response (200 OK)
{ "success": true, "message": "Policy deleted successfully" }
POST /api/policies/reorder สลับลำดับการทำงานหลังผู้ใช้ทำ Drag & Drop จัดเรียงกฎใหม่
[ คลิกดูรายละเอียด ]
Request Body (JSON)
{ "ordered_ids": [2, 1, 3] // ลิสต์ ID ที่เรียงลำดับใหม่แล้ว }
Response (200 OK)
{ "success": true, "message": "Policies reordered successfully" }
POST /api/policies/apply สั่งให้ Backend นำการเปลี่ยนแปลงทั้งหมดไปเขียนลงในระบบ nftables จริง (Apply Configuration)
[ คลิกดูรายละเอียด ]
Response (200 OK)
{ "success": true, "message": "Successfully reloaded nftables with current rules." }
📡 4. DHCP Server API (การจ่าย IP และผูก MAC Address)
GET /api/dhcp/config ดึงค่ากำหนดค่าของ DHCP Server (Pool, DNS, Lease Time)
[ คลิกดูรายละเอียด ]
Response (200 OK)
{ "enabled": true, "interface": "eth0", "start_ip": "192.168.1.100", "end_ip": "192.168.1.200", "gateway": "192.168.1.1", "netmask": "255.255.255.0", "dns_servers": ["8.8.8.8", "1.1.1.1"], "lease_time_seconds": 86400 }
PUT /api/dhcp/config บันทึกตั้งค่า DHCP Server (พร้อมสั่งระบบให้ restart service)
[ คลิกดูรายละเอียด ]
Request Body (JSON)
{ "enabled": true, "start_ip": "192.168.1.100", "end_ip": "192.168.1.200", "gateway": "192.168.1.1", "netmask": "255.255.255.0", "dns_servers": ["8.8.8.8", "1.1.1.1"], "lease_time_seconds": 86400 }
Response (200 OK)
{ "success": true, "message": "DHCP configurations updated and service restarted." }
GET /api/dhcp/leases ดึงรายการอุปกรณ์ที่กำลังเชื่อมต่อและได้รับแจก IP ไปจากเซิร์ฟเวอร์
[ คลิกดูรายละเอียด ]
Response (200 OK)
[ { "ip_address": "192.168.1.101", "mac_address": "99:88:77:66:55:44", "hostname": "iPhone-13", "expires_in": "11 hours, 45 mins" }, { "ip_address": "192.168.1.102", "mac_address": "AA:BB:CC:DD:EE:FF", "hostname": "Android-SmartTV", "expires_in": "23 hours, 10 mins" } ]
🔑 5. Authentication API (ระบบการเข้าสู่ระบบและระบุตัวตน)
POST /api/auth/login ตรวจสอบชื่อผู้ใช้และรหัสผ่านเพื่อสร้าง Session (บันทึก HttpOnly Cookie)
[ คลิกดูรายละเอียด ]
Request Body (JSON)
{ "username": "admin", "password": "yourpasswordhere" }
Response Headers (Set-Cookie)
Set-Cookie: token=JWT_SESSION_TOKEN; HttpOnly; Secure; SameSite=Strict; Path=/; Max-Age=86400
Response (200 OK)
{ "success": true, "message": "Login successful" }
POST /api/auth/logout ออกจากระบบและทำลาย Cookie Session
[ คลิกดูรายละเอียด ]
Response Headers (Clear-Cookie)
Set-Cookie: token=; HttpOnly; Secure; SameSite=Strict; Path=/; Max-Age=0
Response (200 OK)
{ "success": true, "message": "Logged out successfully" }
📡 6. Real-time Streaming API (SSE) (การส่งสถิติ Real-time ผ่าน Server-Sent Events)
GET /api/system/status/stream สตรีมข้อมูลสถิติ CPU, Memory, Temp, WAN Traffic อย่างต่อเนื่อง (Content-Type: text/event-stream)
[ คลิกดูรายละเอียด ]
Event Stream Format
event: system-metrics data: {"cpu": 14.5, "temp": 48.5, "ram": 15.0, "wan": {"in_bps": 4200000, "out_bps": 1100000}} event: system-metrics data: {"cpu": 12.8, "temp": 48.0, "ram": 15.0, "wan": {"in_bps": 3900000, "out_bps": 950000}}
GET /api/system/logs/stream สตรีมข้อความ Logs ล่าสุดจาก nftables และ kernel syslog แบบเรียลไทม์
[ คลิกดูรายละเอียด ]
Event Stream Format
event: firewall-log data: {"time": "2026-06-05T16:22:01Z", "src_ip": "192.168.1.132", "dst_ip": "203.0.113.5", "proto": "TCP", "port": 23, "action": "DROP"}
📖 7. Objects & Routing API (การจัดการ Address/Service Objects และตาราง Routing)
GET /api/addresses ดึงรายการกลุ่มที่อยู่อินเทอร์เน็ต (Address Objects) ทั้งหมด
[ คลิกดูรายละเอียด ]
Response (200 OK)
[ { "id": 1, "name": "LAN_Network", "type": "subnet", "value": "192.168.1.0/24" }, { "id": 2, "name": "Admin_PC", "type": "ip", "value": "192.168.1.100" } ]
GET /api/services ดึงรายการบริการพอร์ตเครือข่าย (Service Objects) ทั้งหมด
[ คลิกดูรายละเอียด ]
Response (200 OK)
[ { "id": 1, "name": "HTTP", "protocol": "TCP", "port": "80" }, { "id": 2, "name": "SSH", "protocol": "TCP", "port": "22" } ]
GET /api/routes ดึงรายการกฎการเชื่อมเส้นทาง Routing (Static Routes) ทั้งหมดในระบบ
[ คลิกดูรายละเอียด ]
Response (200 OK)
[ { "id": 1, "destination": "0.0.0.0/0", "gateway": "10.0.0.1", "interface": "wlan0", "metric": 100 } ]
