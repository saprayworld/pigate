DHCP Server

บันทึกการทำงานของระบบ DHCP Server (ระบบจ่ายไอพีแอดเดรสอัตโนมัติ) การตั้งค่าหลัก การจองไอพี (Static Reservations) และการดึงข้อมูล Active Leases

● Backend Go 1.22+ SQLite dhcpd / dnsmasq อัปเดต 2026-06-15

Active Leases Sync Flow (OS Lease Logs Parsing)
backend/internal/kernel/mock.go → GetActiveLeases()

บนบอร์ด Raspberry Pi 5 ระบบจะวิเคราะห์ประวัติสัญญาเช่าจริงจากระบบปฏิบัติการตามรูปแบบของดาเอมอนที่ตรวจจับได้:

1 เรียก GET /api/dhcp/leases เข้าหลังบ้าน
2 ตรวจสอบไฟล์ประวัติสัญญา: /var/lib/misc/dnsmasq.leases หรือ /var/lib/dhcp/dhcpd.leases
3 แยกฟิลด์ข้อมูลด้วย Regex/String Parsers เพื่อสกัดค่า IP, MAC, และ Hostname อุปกรณ์
4 ส่งอาเรย์ของข้อมูลกลับไปยัง client เพื่อแสดงผลบนหน้าจอแอดมินพอร์ทัลทันที

REST API Endpoints
backend/internal/api/router.go · handlers.go
MethodPathHandlerหน้าที่
GET /api/dhcp/config HandleGetDHCPConfig ดึงการตั้งค่า DHCP หลักของระบบ
PUT /api/dhcp/config HandleUpdateDHCPConfig บันทึกการตั้งค่า DHCP หลักลงฐานข้อมูล SQLite
GET /api/dhcp/reservations HandleGetDHCPReservations เรียกรายการอุปกรณ์ที่จองไอพีทั้งหมด
POST /api/dhcp/reservations HandleCreateDHCPReservation จองไอพีอุปกรณ์ใหม่ (ป้องกันการกรอก MAC ซ้ำ)
PUT /api/dhcp/reservations/{id} HandleUpdateDHCPReservation แก้ไขการจองไอพีของอุปกรณ์
DELETE /api/dhcp/reservations/{id} HandleDeleteDHCPReservation ลบข้อมูลการจองไอพีของอุปกรณ์
GET /api/dhcp/leases HandleGetDHCPLeases ดึงตารางผู้ใช้งานที่กำลังเชื่อมต่อรับไอพีแบบ Active Leases
POST /api/dhcp/apply HandleApplyDHCP เขียนไฟล์ config และสั่ง restart daemon จ่ายไอพีของ OS จริง

ข้อควรระวัง
⚠️
การตั้งอินเตอร์เฟสขาจ่ายไอพี
อินเตอร์เฟสขาที่จะเปิดจ่ายไอพี (เช่น eth0) จะต้องเป็นแบบ **Static IP** เท่านั้น หากการ์ดเครือข่ายตั้งค่ารับไอพีแบบ DHCP ตัวดาเอมอน DHCP Server จะไม่ยอมประมวลผลการทำงาน
⚠️
ช่วงขนาดของ IP Pool (Start IP ↔ End IP)
ช่วงแอดเดรสสำหรับแจกจ่ายจะต้องอยู่ในวง Subnet และ Netmask เดียวกันกับไอพีของอินเตอร์เฟสขานั้น หากไอพีพอร์ตแลนคือ 192.168.1.1/24 ช่วงจ่ายไอพีจะต้องกรอกระหว่าง 192.168.1.2 ถึง 192.168.1.254

Kernel Integration (Production)

ในโหมดระบบทำงานจริง บนบอร์ด Pi Gate จะเขียนทับข้อมูลลงไฟล์ config ของบริการจ่าย IP และสั่ง Restart Service:

dhcpd.conf block preview/etc/dhcp/dhcpd.conf
# Configuration generated dynamically by PiGate Backend subnet 192.168.1.0 netmask 255.255.255.0 { range 192.168.1.100 192.168.1.200; option routers 192.168.1.1; option domain-name-servers 8.8.8.8, 1.1.1.1; default-lease-time 86400; } # Static Reservations mapping from SQL database host device_iphone { hardware ethernet 99:88:77:66:55:44; fixed-address 192.168.1.101; }
