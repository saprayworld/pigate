Helper Functions
backend/internal/db/repository.go

detectAddressingMode(ifaceName, ifaceIndex)

ตรวจสอบว่า interface ใช้ DHCP หรือ Static โดยตรวจ file system ตามลำดับ ถ้าพบไฟล์ที่มีอยู่จริง → คืน "dhcp", fallback → "static"

  • 1/run/dhclient-<iface>.pid— dhclient PID
  • 2/var/run/dhclient.<iface>.pid— dhclient PID (alt path)
  • 3/var/lib/dhcp/dhclient.<iface>.leases— dhclient lease
  • 4/var/lib/dhcpcd/<iface>.lease— dhcpcd5 lease
  • 5/run/systemd/netif/leases/<ifaceIndex>— systemd-networkd
  • 6/var/lib/NetworkManager/dhclient-<iface>.conf— NetworkManager
  • 7/run/NetworkManager/dhcp-<iface>.conf— NetworkManager (runtime)

getGatewayForInterface(ifaceName)

อ่าน /proc/net/route → กรอง row ที่ Iface == ifaceName AND Destination == 00000000 → แปลง Gateway hex (little-endian) เป็น IP

Format/proc/net/route
Iface Destination Gateway Flags RefCnt Use Metric Mask MTU Window IRTT eth1 00000000 0101A8C0 0003 0 0 100 00000000 0 0 0 ↑ Default ↑ Little-endian hex → 192.168.1.1

getInterfaceSpeed(ifaceName) → string

อ่าน /sys/class/net/<iface>/speed (หน่วย Mbps) แปลงเป็น human-readable

ค่าใน sysfsผลลัพธ์
10001 Gbps
100100 Mbps
-1 หรือ errorunknown (wireless ปกติจะได้ค่านี้)

REST API Endpoints

🔐
Authentication Required
ทุก endpoint ต้องผ่าน session cookie หรือ Authorization: Bearer <token>
MethodPathHandlerหน้าที่
GET /api/interfaces HandleGetInterfaces ดึง list ทั้งหมด (trigger OS sync)
PUT /api/interfaces/{id} HandleUpdateInterface แก้ไขค่า interface

HandleToggleInterface — ลำดับการทำงาน

1 ดึง interface จาก DB ตรวจว่ามีอยู่จริง
2 คำนวณ nextStatus → up ↔ down
3 เรียก kernel.NetworkManager.ToggleInterface(name, isUp) → OS action จริง
4 อัปเดต status ใน DB ผ่าน repo.ToggleInterfaceStatus()
5 คืน interface object ที่อัปเดตแล้วกลับไปยัง client

ข้อควรระวัง
⚠️
addressing_mode ไม่ sync อัตโนมัติ
ค่า addressing_mode ในกรณี UPDATE จะไม่ถูก overwrite โดย OS sync — เจตนาเพื่อรักษาการตั้งค่าของ user หากผู้ใช้บันทึก static ไว้แต่ OS ถูกเปลี่ยน → ค่า DB จะยังเป็น static จนกว่าจะ UPDATE ผ่าน API
⚠️
speed อาจเป็น "unknown" สำหรับ wireless
/sys/class/net/wlan0/speed มักคืน -1 เพราะความเร็ว Wi-Fi ขึ้นอยู่กับสภาพสัญญาณ ณ ขณะนั้น ไม่ใช่ค่าคงที่
⚠️
net.FlagUp ≠ Administratively Down
FlagUp อ่านจาก kernel IFF_UP flag เท่านั้น — การรัน nmcli connection down อาจไม่เปลี่ยน IFF_UP เพราะ NM deactivate เฉพาะ connection profile ไม่ได้สั่ง ip link set down เสมอไป ควรใช้ API POST /api/interfaces/{id}/toggle แทน เพื่อให้ kernel state และ DB sync กัน
🔐
Production ต้องการ Linux Capabilities
RealNetwork (netlink) ต้องการ cap_net_admin บน binary ก่อนใช้งาน Production:
sudo setcap cap_net_admin,cap_net_raw+ep ./pigate-backend
