Frontend Technology Stack

เพื่อลดภาระประมวลผลบนตัวบอร์ด Raspberry Pi ให้เหลือน้อยที่สุด หน้าบ้านถูกออกแบบเป็น Single Page Application (SPA) เพื่อให้ประมวลผลบนฝั่ง Client:

⚛️
React 19 & Vite
ช่วยจัดการ UI State ที่มีความซับซ้อน รวดเร็ว มีระบบ Lifecycle ที่นิ่ง และทำงานสอดประสานร่วมกับระบบ Drag & Drop ยุคใหม่ได้เต็มประสิทธิภาพ
🎨
Tailwind CSS v4
ช่วยกำหนด Style ด้วย Utility-first CSS ทำให้เว็บมีขนาดเล็ก โหลดเร็ว ปรับแต่งธีม Dark/Light mode และดีไซน์ Premium Flat ได้อย่างยืดหยุ่น
🧩
shadcn/ui
คอมโพเนนต์ Primitives (Radix UI) ที่ copy-paste โค้ดลงโปรเจกต์โดยตรง ช่วยลด Package Bloat และเปิดสิทธิ์ให้แก้ไขสไตล์ระดับล่างได้อิสระ
📈
Recharts & dnd-kit
@dnd-kit ใช้ทำ Drag & Drop สลับจัดลำดับกฎไฟร์วอลล์ และ Recharts สำหรับวาดกราฟสถิติทราฟฟิก WAN แบบเรียลไทม์
💡
Deployment Pattern — go:embed
เมื่อหน้าบ้านถูกคอมไพล์ผ่านคำสั่ง yarn build ไฟล์ปลายทาง HTML, CSS, JS จะถูกฝังเข้าไปเป็นส่วนหนึ่งของไฟล์แอปพลิเคชันหลังบ้าน Go ผ่านระบบ go:embed ทำให้ติดตั้งและส่งมอบงานในลักษณะ Single Binary ไฟล์เดียวจบ

Backend Technology Stack (Go/Golang)

หลังบ้านพัฒนาด้วยภาษา Go (Golang) เพื่อความเร็ว ความเสถียรสูงสุด และป้องกันปัญหา Supply Chain Attack:

⚙️
Go Standard Library (net/http)
หลีกเลี่ยงการพึ่งพาเฟรมเวิร์กขนาดใหญ่ภายนอก ช่วยลดปริมาณโมดูลเสริมให้เหลือเกือบ 0% ลดความเสี่ยงช่องโหว่ทางห่วงโซ่อุปทานลงอย่างสมบูรณ์
🗄️
modernc.org/sqlite
ตัวเชื่อมต่อ SQLite ที่เขียนด้วยภาษา Go ล้วน (Pure Go / CGO-free) ทำให้สามารถ Cross-compile ข้ามแพลตฟอร์มได้โดยไม่ต้องพึ่ง C Compiler
4
Kernel Integration & Privilege Separation

ระบบควบคุมระดับ Gateway จำเป็นต้องเปลี่ยนค่า Firewall และสถานะอินเทอร์เฟซ ซึ่งต้องใช้สิทธิ์ระดับสูง แต่เพื่อป้องกันอันตรายจากการครอบงำระบบ จึงออกแบบโครงสร้างความปลอดภัยไว้ดังนี้:

4.1 Direct Socket Interaction (หลีกเลี่ยง Command Injection)

การใช้คำสั่งเชลล์มีความเสี่ยงต่อช่องโหว่ Command Injection หากกรองอักขระอินพุตไม่รัดกุม ระบบ PiGate จึงเลือกใช้ Pure socket สื่อสารกับ Linux Kernel โดยตรง:

  • 📁 Firewall (nftables): สื่อสารผ่าน Netlink Socket ด้วย github.com/google/nftables สำหรับสร้าง ลบ และดึงข้อมูลกฎตาราง Firewall
  • 📁 Network & Routing: สื่อสารผ่าน Netlink ด้วย github.com/vishvananda/netlink สำหรับการปรับปรุง Interface IP, Gateway, และ Routing Table

4.2 Linux Capabilities (การควบคุมขอบเขตความปลอดภัย)

แอปพลิเคชันหลังบ้าน Go จะไม่ได้ถูกรันภายใต้สิทธิ์ root ตรงๆ แต่จะถูกจำกัดสิทธิ์ในระดับ User ธรรมดา และใช้วิธีผูก Linux Capabilities ให้ทำงานเฉพาะทางได้เท่านั้น:

bashPrivilege assignment
sudo setcap cap_net_admin,cap_net_raw+ep ./pigate-backend

- cap_net_admin: สิทธิ์ตั้งค่า Network Interface, IP, Routes และ Firewall Rules
- cap_net_raw: สิทธิ์ในการสร้าง Raw Sockets (ใช้ในการทำ Ping หรือสอดส่องแพ็กเก็ต)

5
Supply Chain Attack Prevention
🔒
Strict Dependency Pinning
การใช้ระบบ go.sum ในภาษา Go เพื่อบันทึกค่า Cryptographic Hash ของทุกโมดูล ป้องกันการแอบสอดไส้โค้ดจากต้นทางผู้พัฒนาภายนอกระหว่างการคอมไพล์
🛡️
Official & Core Modules Only
คัดกรองเฉพาะไลบรารีระดับแกนหลักที่มีผู้พัฒนาอย่างเป็นทางการและน่าเชื่อถือสูงเท่านั้น (เช่น golang.org/x/... หรือ github.com/google/...)
🚀
No Dynamic Code Evaluation
ภาษา Go ไม่มีฟังก์ชันการรันโค้ดสดผ่าน string (เช่น eval() ใน JavaScript) ทำให้ปลอดภัยจากช่องโหว่ Dynamic Code Injection โดยปริยาย
6
Authentication & Access Control

ระบบความมั่นคงปลอดภัยของการจำกัดสิทธิ์เข้าใช้งาน API:

  • 🔑 Password Hashing: เข้ารหัสรหัสผ่านผู้ดูแลระบบด้วยความปลอดภัยสูงผ่าน bcrypt
  • 🍪 Session Control: บันทึก JWT/Session Token บนเว็บบราวเซอร์ผ่าน HttpOnly, Secure, SameSite=Strict Cookies เพื่อบล็อกการเข้าถึงจาก JavaScript (ป้องกันการถูกขโมยผ่านช่องโหว่ XSS)
  • ⏳ Rate Limiting: ป้องกันการสุ่มรหัสผ่านผ่าน Brute-Force Attacks ในหน้า Login ด้วยระบบ Rate Limiter (Token Bucket Algorithm)
7
Real-Time Data Streaming

การสตรีมค่าสถานะกราฟ WAN Bandwidth, ทรัพยากรระบบ CPU/RAM/Temp และ Block Logs ปรากฏแบบเรียลไทม์บนหน้าหลัก Dashboard:

📡
Server-Sent Events (SSE)
ใช้ SSE ซึ่งเป็นโปรโตคอลทิศทางเดียว (Server-to-Client) รันผ่าน HTTP ปกติ ทำงานได้เบาและกินทรัพยากรน้อยกว่าการเชื่อมต่อผ่าน WebSockets
⚡
Zero Third-Party Library
สตรีมโดยตรงผ่าน http.ResponseWriter มาตรฐานและมีระบบจับช่องสัญญาณหลุดของบราวเซอร์เพื่อเคลียร์หน่วยความจำโดยอัตโนมัติ
8
SD Card Preservation (การถนอมการเขียน MicroSD)
⚠️
ข้อจำกัดของ MicroSD Card บน Raspberry Pi
บอร์ด Raspberry Pi มักรัน OS ผ่าน MicroSD Card ซึ่งมีรอบการเขียนจำกัด การเขียนไฟล์ล็อกขนาดใหญ่ลงดิสก์ SQLite อย่างต่อเนื่องจะทำให้การ์ดเสื่อมสภาพและชำรุดเสียหายได้อย่างรวดเร็ว

ระบบป้องกันปัญหา SD Card พังเร็วด้วยโซลูชัน:

  • 💾 In-Memory Circular Buffer: บันทึกล็อกการดักจับแพ็กเก็ตล่าสุด (Firewall Block Logs) ลงใน Ring Buffer บนแรมของแอปพลิเคชัน Go แทนการบันทึกทับลงฐานข้อมูล SQLite
  • 💾 Virtual Filesystem: เขียนไฟล์ล็อกทั่วไปลงพาร์ทิชันชั่วคราว เช่น /run/ หรือ /tmp/ (ซึ่งทำงานบนหน่วยความจำแรม) เพื่อเลี่ยงการกดเขียนข้อมูลจริงลงดิสก์การ์ด
9
Network Configuration Mechanism (D-Bus Protocol)

การควบคุมพอร์ตการ์ดแลนและ Wi-Fi ในระบบปฏิบัติการ Raspberry Pi OS รุ่นใหม่จะรันผ่านบริการ NetworkManager:

🖧
D-Bus IPC Socket Connection
แทนการใช้คำสั่ง shell-wrap เช่น nmcli โค้ดหลังบ้าน Go จะสื่อสารผ่าน D-Bus Socket ดั้งเดิมของ OS (ใช้ godbus/dbus) เพื่อความปลอดภัยและแม่นยำสูง
⚡
Fast & Precise Results
ช่วยให้อ่านข้อมูลความแรงสัญญาณ Wi-Fi, จัดการ SSID และตั้งค่า addressing mode ได้เสถียรกว่าการประมวลผล text จาก command line
